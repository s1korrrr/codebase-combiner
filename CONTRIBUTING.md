# Contributing

Thanks for contributing! This project aims to keep changes small, well-tested, and easy to review.

## Getting started

- Read `INSTALL.md` for setup.
- Run tests and lint before submitting:
  - `npm test`
  - `npm run lint`
  - `npm run format:check`
  - `cd SwiftExplorerApp && swift test`
  - `swiftformat --lint .`

## Pull requests

- Keep PRs focused and minimal.
- Include tests for new or changed behavior.
- Update documentation if user-visible behavior changes.
- Ensure all quality gates are green.

## Commit messages

- Use clear, descriptive messages that state intent and scope.

## Reporting issues

- Include reproduction steps and expected vs actual behavior.
- Attach logs or screenshots if relevant.
